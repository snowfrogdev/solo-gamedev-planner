# Law of Similarity

The human eye tends to perceive similar elements in a design as a complete picture, shape, or group, even if those elements are separated.

## Description

The Law of Similarity is a Gestalt principle stating that elements sharing visual characteristics — such as color, shape, size, orientation, or texture — are perceived as belonging to the same group and having related functionality. This principle allows designers to communicate functional relationships through visual consistency alone. When interactive elements look the same, users assume they behave the same; when non-interactive elements look like interactive ones, users become confused and frustrated.

## Key Takeaways

- Elements that are visually similar will be perceived as related.
- Color, shape, size, orientation, and movement can signal that elements belong to the same group and likely share a common meaning or functionality.
- Ensure that links and navigation systems are visually differentiated from normal text elements.
- Visual similarity creates powerful user expectations about functionality — elements appearing identical should perform similarly.

## Origins

The principles of grouping (Gestalt laws of grouping) are a set of principles in psychology, first proposed by Gestalt psychologists to account for the observation that humans naturally perceive objects as organized patterns and objects — a principle known as Prägnanz. Gestalt psychologists argued that these principles exist because the mind has an innate disposition to perceive patterns in the stimulus based on certain rules. Similarity is one of the five foundational categories: Proximity, Similarity, Continuity, Closure, and Connectedness.

## Practical Implications for UI Design

- **Consistent interactive styling**: All clickable elements should share visual characteristics (color, shape, cursor style) that distinguish them from non-interactive elements. This is the most fundamental application of the similarity principle.
- **Color strategy for CTAs**: Use a consistent link color to signal clickability across all pages. Reserve a distinct color for primary calls-to-action to differentiate them from secondary buttons. Never apply interactive colors to non-clickable elements.
- **Shape consistency in navigation**: Apply matching shapes (like rounded rectangles) to related navigation items to strengthen their perceived grouping. Use distinct icon shapes to help users quickly scan and differentiate list items.
- **Size hierarchy**: Display related content at consistent sizes to communicate equal importance. Vary size strategically to distinguish promotional content from regular listings.
- **Avoid false similarity**: Non-interactive elements styled to look like buttons or links create false expectations and frustrate users. This includes underlined non-link text, colored boxes that look clickable, and similarly-sized ads mixed with content (triggering banner blindness).
- **Differentiate categories visually**: When presenting items from different categories or types, give each category a distinct visual treatment so users can quickly identify and filter.

## Further Reading Insights

**Nielsen Norman Group (Aurora Harley)** — Similarity is one of the most powerful grouping principles because it operates through multiple visual channels simultaneously. Key applications include consistent link colors for signaling clickability, matching shapes for navigation items, and strategic size variation for content hierarchy. Critical warning: visual similarity creates powerful expectations — elements appearing identical must perform identically, or users face confusion.

**Smashing Magazine (Steven Bradley)** — Similarity works alongside proximity and common region to create visual hierarchy in web design. Elements sharing visual characteristics (color, shape, size, texture) are perceived as related regardless of their spatial position. Apply consistent link styling to signal that multiple text elements share the same function. Use color coding in data visualizations to connect related information across different areas of a display.

**Chris Butler** — The principle of similarity in design works by creating visual "categories" that the brain uses for rapid classification. Consistent use of visual similarity reduces the cognitive effort required to understand interface organization, while inconsistency forces conscious processing of each individual element.
